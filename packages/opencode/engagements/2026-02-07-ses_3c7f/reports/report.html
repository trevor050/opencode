HIGH SEVERITY FINDINGS

FND-SES8G4F0Z4: Redis 8.0.0 Without Authentication
HIGH

Asset: 127.0.0.1:6379 | Confidence: 0.95

Executive Summary

Redis version 8.0.0 is running without authentication. Any local process can access the database, read cached data, modify values, and issue administrative commands. This creates a critical data security vulnerability if other local services are compromised.

Technical Details

  • Service: Redis Database Server v8.0.0
  • Binding: IPv4 (127.0.0.1:6379) + IPv6 ([::1]:6379)
  • Authentication: NONE (no requirepass configured)
  • Validation: redis-cli INFO received without authentication error

Business Impact

  • Confidentiality: HIGH — Any local process can read cached data
  • Integrity: HIGH — Unauthorized modification of values possible
  • Availability: HIGH — Commands like FLUSHDB can destroy data
  • Escalation: CRITICAL — If any application is compromised, Redis becomes attack vector

Remediation

Phase 1 (IMMEDIATE): Enable authentication 
openssl rand -hex 32  # Generate strong password
redis-cli CONFIG SET requirepass "[your-password]"
# Edit /opt/homebrew/etc/redis.conf and add:
# requirepass [your-password]
brew services restart redis
Effort: 15 minutes | Risk Reduction: 95%
Timeline: Complete TODAY
FND-QKC74KB5QE: Apple Diagnostics Service (rapportd) on All Network Interfaces
HIGH

Asset: *.55992 | Confidence: 0.80

Executive Summary

Apple's diagnostic reporting service (rapportd) listens on all network interfaces, making it accessible to any device on the local network. This service exposes system diagnostics and crash reports to LAN-based attackers.

Technical Details

  • Service: Apple Diagnostic Reporting (rapportd)
  • Port: 55992/TCP
  • Binding: Wildcard (all interfaces)
  • Accessibility: Any device on 192.168.1.0/24 can connect

Business Impact

  • Information Disclosure: MEDIUM — System diagnostics exposed to LAN
  • Lateral Movement: MEDIUM — Attackers can enumerate system state
  • Vulnerability Exposure: MEDIUM — If rapportd has CVEs, remote exploitation possible
  • Network Risk: SIGNIFICANT — 4 other devices detected on network

Remediation

Phase 1 (IMMEDIATE): Disable diagnostic service 
# Disable Analytics in System Preferences:
# Settings → Security & Privacy → Analytics → Uncheck all

# Unload rapportd service
sudo launchctl unload -w /System/Library/LaunchDaemons/com.apple.rapportd.plist

# Verify it's disabled
sudo launchctl list | grep rapportd  # Should return no results
Effort: 5 minutes | Risk Reduction: 80%
Timeline: Complete TODAY
